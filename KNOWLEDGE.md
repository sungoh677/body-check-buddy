# 프로젝트 지식 (Project Knowledge)

## 핵심 가치 (Core Values)
1. **20초 점검 (20-Second Check)** [핵심]
   - 사용자가 20초 이내에 자신의 상태를 빠르고 직관적으로 점검하고 기록할 수 있도록 UI/UX를 설계한다. (빠른 입력 및 피드백)
2. **다크 모드 유지 (Maintain Dark Mode)** [핵심]
   - 헬스케어 서비스다운 깔끔하고 전문적인 톤앤매너를 유지하기 위해, 현재 적용된 다크 모드 테마를 앱 전반에 걸쳐 일관되게 유지한다.
3. **안전성 및 확인 (Safety & Confirmation)**
   - 중요한 파일(DB 설정, 핵심 환경변수 등)을 수정하거나 삭제하기 전에는 반드시 사용자에게 먼저 묻고 진행한다.

## 기술 스택 (Tech Stack)
- **프론트엔드**: React, TypeScript, Tailwind CSS, Vite, shadcn-ui
- **백엔드/DB**: Supabase
- **언어 제약**: `any` 타입 사용은 엄격히 금지하며, 타입 안정성을 최우선으로 확보한다. 모든 답변과 주석은 한국어로 작성한다.

## 아키텍처 가이드 (Architecture Guide)
- **역할 분리 (Clean Architecture 지향)**: 파일 하나에 모든 로직을 몰아넣지 않고, UI, 상태 관리, API 호출(Supabase) 등의 역할을 명확히 분리하여 모듈화한다.
- **확장성 고려**: 향후 AI 코칭, 1:1 챌린지, 소셜 기능(뉴스, 댓글 등) 확장을 염두에 두고 재사용 가능한 컴포넌트와 함수로 분리한다.
